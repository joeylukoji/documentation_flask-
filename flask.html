1. Introduction

Flask est un micro-framework Python pour créer des applications web. Il est léger, flexible et idéal pour apprendre le développement web.

Concepts clés :

Concept Description
Route URL associée à une fonction Python
Template Fichier HTML généré dynamiquement (Jinja2)
Request Données envoyées par le navigateur (GET, POST…)
Session Données stockées côté serveur pour un utilisateur connecté
CRUD Create, Read, Update, Delete

5.2 Route de connexion — dans `app.py`

from flask import Flask, render_template, request, redirect, url_for, session, flash

app = Flask(__name__)
app.secret_key = "changez-cette-cle-secrete-en-production"

# Utilisateurs de démonstration (en production : base de données + hash)
USERS = {
    "admin@example.com": {"password": "admin123", "name": "Administrateur"},
    "user@example.com": {"password": "user123", "name": "Utilisateur"},
}

@app.route("/login", methods=["GET", "POST"])
def login():
    if request.method == "POST":
        email = request.form.get("email", "").strip()
        password = request.form.get("password", "")

        user = USERS.get(email)
        if user and user["password"] == password:
            session["user_email"] = email
            session["user_name"] = user["name"]
            flash("Connexion réussie.", "success")
            return redirect(url_for("dashboard"))

        flash("Email ou mot de passe incorrect.", "error")

    return render_template("login.html")

@app.route("/logout")
def logout():
    session.clear()
    flash("Vous êtes déconnecté.", "success")
    return redirect(url_for("login"))

5.3 Points importants du formulaire de connexion

Élément Rôle
method="POST" Envoie les données de façon sécurisée (pas dans l'URL)
name="email" Clé utilisée avec request.form.get("email")
required Validation HTML côté navigateur
session[...] Mémorise l'utilisateur connecté
redirect() Redirige après connexion réussie

6. Sessions et protection des routes

6.1 Décorateur pour protéger les pages

from functools import wraps

def login_required(f):
    @wraps(f)
    def decorated_function(*args, **kwargs):
        if "user_email" not in session:
            flash("Veuillez vous connecter.", "error")
            return redirect(url_for("login"))
        return f(*args, **kwargs)
    return decorated_function

@app.route("/dashboard")

def dashboard():
    return render_template("dashboard.html", name=session.get("user_name"))

7.1 Modèle et base de données

Avant le CRUD, il faut définir comment les données sont stockées. En production, on utilise une base de données, pas une simple liste Python.

Liste Python vs Base de données

Liste managed_users = [] Base SQLite + modèle
Données au redémarrage Perdues Conservées dans app.db
Structure {"name": "Alice"} Colonnes : id, name, email…
Recherche Boucle for User.query.filter_by(...)

Qu'est-ce qu'un modèle ?

Un modèle est une classe Python = une table SQL. Chaque attribut = une colonne. Chaque objet User(...) = une ligne.

class User              →  Table "user"
    .id                 →  colonne id (clé primaire)
    .name               →  colonne name
    .email              →  colonne email
    .role               →  colonne role
    .password_hash      →  colonne password_hash

Installation

pip install flask-sqlalchemy

Configuration — `app.py`

from flask import Flask
from flask_sqlalchemy import SQLAlchemy

app = Flask(__name__)
app.config["SQLALCHEMY_DATABASE_URI"] = "sqlite:///app.db"
app.config["SQLALCHEMY_TRACK_MODIFICATIONS"] = False

db = SQLAlchemy(app)

Modèle `User` — `models.py`

from app import db

class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(100), nullable=False)
    email = db.Column(db.String(120), unique=True, nullable=False)
    role = db.Column(db.String(20), default="user")
    password_hash = db.Column(db.String(200), nullable=False)

    def __repr__(self):
        return f"<User {self.email}>"
Colonne Type Description
id Integer, primary_key=True Identifiant unique auto-généré
name String(100), nullable=False Nom obligatoire
email String(120), unique=True Email unique (pas de doublon)
role String(20) Rôle : user, admin, etc.
password_hash String(200) Mot de passe hashé (jamais en clair)

Créer la table en base

with app.app_context():
    db.create_all()

Exécuter une fois → crée le fichier app.db et la table user.

Opérations sur le modèle (sans route HTTP)

from werkzeug.security import generate_password_hash, check_password_hash

# CREATE
user = User(name="Alice", email="alice@example.com",
            role="user", password_hash=generate_password_hash("secret"))
db.session.add(user)
db.session.commit()

# READ — tous
users = User.query.all()

# READ — un seul par id
user = db.session.get(User, 1)

# READ — par email
user = User.query.filter_by(email="alice@example.com").first()

# UPDATE
user.name = "Alice Martin"
db.session.commit()

# DELETE
db.session.delete(user)
db.session.commit()

commit() est obligatoire après add(), modification ou delete() — sinon rien n'est écrit en base.

7.4 CREATE — Créer un utilisateur

Route :

from werkzeug.security import generate_password_hash
from models import User

@app.route("/users/create", methods=["GET", "POST"])
@login_required
def create_user():
    if request.method == "POST":
        name = request.form.get("name", "").strip()
        email = request.form.get("email", "").strip()
        role = request.form.get("role", "user")
        password = request.form.get("password", "")

        errors = []
        if not name:
            errors.append("Le nom est obligatoire.")
        if not email:
            errors.append("L'email est obligatoire.")
        if User.query.filter_by(email=email).first():
            errors.append("Cet email existe déjà.")

        if errors:
            for error in errors:
                flash(error, "error")
            return render_template("users/create.html", name=name, email=email, role=role)

        user = User(
            name=name,
            email=email,
            role=role,
            password_hash=generate_password_hash(password or "changeme"),
        )
        db.session.add(user)
        db.session.commit()
        flash(f"Utilisateur « {name} » créé avec succès.", "success")
        return redirect(url_for("list_users"))

    return render_template("users/create.html")

7.5 UPDATE — Modifier un utilisateur

Route :

@app.route("/users/<int:user_id>/edit", methods=["GET", "POST"])
@login_required
def edit_user(user_id):
    user = db.session.get(User, user_id)
    if not user:
        flash("Utilisateur introuvable.", "error")
        return redirect(url_for("list_users"))

    if request.method == "POST":
        name = request.form.get("name", "").strip()
        email = request.form.get("email", "").strip()
        role = request.form.get("role", "user")

        errors = []
        if not name:
            errors.append("Le nom est obligatoire.")
        if not email:
            errors.append("L'email est obligatoire.")
        existing = User.query.filter_by(email=email).first()
        if existing and existing.id != user_id:
            errors.append("Cet email est déjà utilisé.")

        if errors:
            for error in errors:
                flash(error, "error")
            return render_template("users/edit.html", user=user)

        user.name = name
        user.email = email
        user.role = role
        db.session.commit()
        flash("Utilisateur mis à jour.", "success")
        return redirect(url_for("show_user", user_id=user_id))

    return render_template("users/edit.html", user=user)

7.6 DELETE — Supprimer un utilisateur

Route :

@app.route("/users/<int:user_id>/delete", methods=["POST"])
@login_required
def delete_user(user_id):
    user = db.session.get(User, user_id)
    if not user:
        flash("Utilisateur introuvable.", "error")
        return redirect(url_for("list_users"))

    name = user.name
    db.session.delete(user)
    db.session.commit()
    flash(f"Utilisateur « {name} » supprimé.", "success")
    return redirect(url_for("list_users"))

Note : La suppression utilise POST (et non GET) pour éviter qu'un simple lien ne supprime des données par accident.

8.4 Validation côté serveur

Toujours valider côté serveur, même si required est présent en HTML :

def validate_user_form(name, email, role):
    errors = []

    if not name or len(name) < 2:
        errors.append("Le nom doit contenir au moins 2 caractères.")

    if not email or "@" not in email:
        errors.append("Email invalide.")

    if role not in ("user", "admin"):
        errors.append("Rôle invalide.")

    return errors

10. Exemple complet

Voici l'application Flask complète avec modèle de base de données :

app.py

from functools import wraps
from flask import Flask, render_template, request, redirect, url_for, session, flash
from flask_sqlalchemy import SQLAlchemy
from werkzeug.security import generate_password_hash, check_password_hash

app = Flask(__name__)
app.secret_key = "changez-cette-cle-secrete-en-production"
app.config["SQLALCHEMY_DATABASE_URI"] = "sqlite:///app.db"
app.config["SQLALCHEMY_TRACK_MODIFICATIONS"] = False

db = SQLAlchemy(app)


# --- Modèle (table user en base) ---
class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(100), nullable=False)
    email = db.Column(db.String(120), unique=True, nullable=False)
    role = db.Column(db.String(20), default="user")
    password_hash = db.Column(db.String(200), nullable=False)


def login_required(f):
    @wraps(f)
    def decorated(*args, **kwargs):
        if "user_email" not in session:
            flash("Veuillez vous connecter.", "error")
            return redirect(url_for("login"))
        return f(*args, **kwargs)
    return decorated


# --- Auth ---
@app.route("/")
def index():
    if "user_email" in session:
        return redirect(url_for("dashboard"))
    return redirect(url_for("login"))


@app.route("/login", methods=["GET", "POST"])
def login():
    if request.method == "POST":
        email = request.form.get("email", "").strip()
        password = request.form.get("password", "")
        user = User.query.filter_by(email=email).first()
        if user and check_password_hash(user.password_hash, password):
            session["user_email"] = email
            session["user_name"] = user.name
            flash("Connexion réussie.", "success")
            return redirect(url_for("dashboard"))
        flash("Email ou mot de passe incorrect.", "error")
    return render_template("login.html")


@app.route("/logout")
def logout():
    session.clear()
    flash("Déconnexion effectuée.", "success")
    return redirect(url_for("login"))


@app.route("/dashboard")
@login_required
def dashboard():
    return render_template("dashboard.html", name=session.get("user_name"))


# --- CRUD ---
@app.route("/users")
@login_required
def list_users():
    users = User.query.all()
    return render_template("users/list.html", users=users)


@app.route("/users/create", methods=["GET", "POST"])
@login_required
def create_user():
    if request.method == "POST":
        name = request.form.get("name", "").strip()
        email = request.form.get("email", "").strip()
        role = request.form.get("role", "user")
        password = request.form.get("password", "")

        if not name or not email or not password:
            flash("Tous les champs sont obligatoires.", "error")
            return render_template("users/create.html", name=name, email=email, role=role)

        if User.query.filter_by(email=email).first():
            flash("Cet email existe déjà.", "error")
            return render_template("users/create.html", name=name, email=email, role=role)

        user = User(name=name, email=email, role=role,
                    password_hash=generate_password_hash(password))
        db.session.add(user)
        db.session.commit()
        flash("Utilisateur créé.", "success")
        return redirect(url_for("list_users"))

    return render_template("users/create.html")


@app.route("/users/<int:user_id>/edit", methods=["GET", "POST"])
@login_required
def edit_user(user_id):
    user = db.session.get(User, user_id)
    if not user:
        flash("Utilisateur introuvable.", "error")
        return redirect(url_for("list_users"))

    if request.method == "POST":
        user.name = request.form.get("name", "").strip()
        user.email = request.form.get("email", "").strip()
        user.role = request.form.get("role", "user")
        db.session.commit()
        flash("Utilisateur mis à jour.", "success")
        return redirect(url_for("list_users"))

    return render_template("users/edit.html", user=user)


@app.route("/users/<int:user_id>/delete", methods=["POST"])
@login_required
def delete_user(user_id):
    user = db.session.get(User, user_id)
    if user:

        flash("Utilisateur supprimé.", "success")
    return redirect(url_for("list_users"))

Schéma des routes

GET  /                    → Redirection login ou dashboard
GET  /login               → Formulaire de connexion
POST /login               → Traitement connexion
GET  /logout              → Déconnexion
GET  /dashboard           → Tableau de bord (protégé)

GET  /users               → Liste (Read)
GET  /users/create        → Formulaire création (Create)
POST /users/create        → Traitement création
GET  /users/<id>          → Détail (Read)
GET  /users/<id>/edit     → Formulaire modification (Update)
POST /users/<id>/edit     → Traitement modification
POST /users/<id>/delete   → Suppression (Delete)

Bonnes pratiques

  1. Ne jamais stocker les mots de passe en clair — utiliser werkzeug.security (generate_password_hash, check_password_hash).
  2. Toujours valider côté serveur — la validation HTML peut être contournée.
  3. Utiliser POST pour les actions destructives (suppression, modification).
  4. Changer secret_key en production et ne jamais la committer.
  5. Passer à une base de données (SQLite, PostgreSQL) pour un vrai projet — voir Flask-SQLAlchemy.
